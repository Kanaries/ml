# HDBSCAN in JavaScript

## Algorithm overview

HDBScan finds clusters with varying density and identifies noise points without forcing every point into a cluster.

This algorithm is especially useful when:

- Cluster shapes are irregular and density changes across regions.
- Outlier/noise detection is important for your downstream workflow.
- You do not know an exact cluster count in advance.

## JavaScript implementation

@kanaries/ml brings HDBSCAN-style density clustering into JavaScript so you can group irregular shapes and detect noise directly in browser-based analysis tools or Node.js data services. That is especially helpful for exploratory products where users upload data and expect clustering to happen immediately in the same environment.

For teams building interactive analytics, this means dense cluster discovery can stay close to the UI rather than being delegated to a separate Python backend for every run.

### Quick JavaScript example

```ts
import { Clusters } from '@kanaries/ml';

const X = [[0, 0], [0.1, 0.2], [5, 5], [5.2, 5.1], [9, 0]];

const model = new Clusters.HDBScan(2, null, 0.4);
const labels = model.fitPredict(X);
```

## Detailed API reference

```ts
constructor(
    min_cluster_size: number = 5,
    min_samples: number | null = null,
    cluster_selection_epsilon: number = 0.5,
    metric: Distance.IDistanceType = 'euclidean'
)
```

`fitPredict(samplesX: number[][]): number[]` returns cluster labels. Noise points are marked as `-1`.

This is a simplified implementation that internally calls DBSCAN using `cluster_selection_epsilon` as the `eps` parameter.

```ts
const hdb = new Clusters.HDBScan(2, null, 0.4);
const labels = hdb.fitPredict(X);
```

### Implementation workflow

1. Scale features so distance comparisons are meaningful.
2. Fit the model and inspect cluster labels plus noise assignments.
3. Use cluster stability and business metrics to tune density settings.

### JavaScript deployment notes

- Use HDBSCAN when cluster count is unknown and you expect some observations to remain noise.
- In browser workflows, reserve it for moderate dataset sizes or run it off the main thread.
- Compare its behavior with DBSCAN when you need a simpler density-based baseline.
