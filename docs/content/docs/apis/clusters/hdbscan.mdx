# HDBSCAN in JavaScript

## Algorithm overview

> **⚠️ Experimental — simplified implementation.** The `HDBScan` class in @kanaries/ml is currently a **DBSCAN-based approximation**, not the full HDBSCAN algorithm. It runs DBSCAN with `cluster_selection_epsilon` as the fixed `eps`, so it does **not** handle clusters of varying density, and `min_cluster_size` does not affect cluster extraction the way it does in scikit-learn's HDBSCAN. If your data has clusters of significantly different densities, results will differ from Python HDBSCAN — consider tuning [DBSCAN](/docs/apis/clusters/dbscan) directly instead. A full HDBSCAN implementation (mutual reachability, MST, hierarchy condensation, stability-based extraction) is under evaluation.

The full HDBSCAN algorithm finds clusters with varying density and identifies noise points without forcing every point into a cluster. The current @kanaries/ml class covers the fixed-density subset of that behavior:

- Outlier/noise detection is important for your downstream workflow.
- You do not know an exact cluster count in advance.
- Your clusters have roughly similar density (a single `eps` fits all of them).

## JavaScript implementation

@kanaries/ml exposes an `HDBScan` class with an HDBSCAN-like constructor signature so code can be ported now and upgraded transparently if a full implementation lands. Internally it delegates to DBSCAN today (see the note above), which keeps density-based noise detection available directly in browser-based analysis tools or Node.js data services.

### Quick JavaScript example

```ts
import { Clusters } from '@kanaries/ml';

const X = [[0, 0], [0.1, 0.2], [5, 5], [5.2, 5.1], [9, 0]];

const model = new Clusters.HDBScan(2, null, 0.4);
const labels = model.fitPredict(X);
```

## Detailed API reference

```ts
constructor(
    min_cluster_size: number = 5,
    min_samples: number | null = null,
    cluster_selection_epsilon: number = 0.5,
    metric: Distance.IDistanceType = 'euclidean'
)
```

`fitPredict(samplesX: number[][]): number[]` returns cluster labels. Noise points are marked as `-1`.

This is a simplified implementation that internally calls DBSCAN using `cluster_selection_epsilon` as the `eps` parameter.

```ts
const hdb = new Clusters.HDBScan(2, null, 0.4);
const labels = hdb.fitPredict(X);
```

### Implementation workflow

1. Scale features so distance comparisons are meaningful.
2. Fit the model and inspect cluster labels plus noise assignments.
3. Use cluster stability and business metrics to tune density settings.

### JavaScript deployment notes

- Use this class when cluster count is unknown, you expect some observations to remain noise, and cluster densities are roughly uniform.
- In browser workflows, reserve it for moderate dataset sizes or run it off the main thread.
- Because the current implementation delegates to DBSCAN, results are equivalent to `DBScan(cluster_selection_epsilon, min_samples)` — do not expect parity with Python HDBSCAN on varying-density data.
